# EchoMinutes Changelog

本文件记录面向用户的版本变化。每个正式版本必须包含与 Git 标签一致的章节，GitHub Release 会直接使用对应章节作为发布说明。

## [Unreleased]

后续变化将在这里记录。

## [1.1.2] - 2026-07-23

### 新增

- 历史会话现在保存录音轨道路径、来源、时间轴偏移和时长；重新打开会话后可以点击任意转写时间戳，从对应麦克风、系统音频或导入 WAV 的位置开始播放。
- 主工作台新增“停止播放”，历史列表显示每个会话的录音轨道数量，并对已移动或删除的录音文件给出明确警告。
- SQLite schema 升级到 v3，旧 v1/v2 数据库会自动创建录音轨道表，不改变已有会话、speaker 或 segment。

### 改进

- 同一会话多次开始录音时使用唯一 WAV 文件名，并按单调时钟把后续录音追加到已有时间轴，不再覆盖同一设备的上一条录音。
- 录音后离线精修保留轨道元数据，并把每条精修结果加上该轨道的时间轴偏移，确保点击时间戳仍映射到正确文件。
- 导入媒体抽取出的 WAV 会随新会话一起登记；数据库优先保存相对路径，使整个便携目录移动后仍可恢复录音关系。

### 修复

- 修复停止采集时先释放 WAV writer、最后一个音频回调可能来不及写入的问题；现在先停止设备，再封口 WAV 并读取准确时长。
- 修复重复录音的实时转写时间重新从 0 秒开始、与旧片段重叠的问题。
- 修复历史会话只剩录音、转写已删空时被当作空白会话丢弃的问题。

### 数据与兼容性

- 删除历史会话只删除 SQLite 中的引用，不自动删除原始 WAV，避免不可逆清理用户录音。
- 录音文件缺失不会阻止历史列表或转写打开；播放时会显示具体文件名并写入本地诊断日志。
- 版本入口提升为 `1.1.2`；正式发布仍统一 App、Core、Updater 与安装器版本。

## [1.1.1] - 2026-07-23

### 新增

- 新增“会议档案”历史会话页面，可查看会话时间、时长、片段数与说话人数，并打开、删除或新建会议。
- 新增可编辑转写工作台：支持搜索、直接修改文本、`Ctrl+Enter` 按光标拆分、合并上一段和删除片段。
- 新增可选的录音后高质量精修：实时转写继续作为预览，停止后可使用当前离线模型重新转写完整音轨；麦克风保持“我”，远端音轨可继续执行说话人分离。
- 新增导入/精修取消按钮，长任务不再只能通过关闭程序中止。
- 新增本地滚动诊断日志，记录版本、配置恢复、导入、会话保存、实时识别与异常信息，默认位于 `data/logs/echo-minutes.log`。
- 新增 Windows、Ubuntu 与 macOS 的 Core/STT CI，以及独立的 Windows WPF 全量构建和烟雾测试。

### 改进

- SQLite schema 升级到 v2：speaker 与 segment 使用会话内复合键，保存时在单个事务中完整同步会话子表。
- 旧数据库会自动迁移，并从现有 segment 快照重建可能已被全局 speaker 主键覆盖的历史说话人。
- 实时音频进入容量固定的单消费者队列；识别跟不上输入时跳过新的转写块，但原始 WAV 录音继续完整保存，避免 `async void` 任务无限积压。
- FFprobe 时长检测改为异步、可取消且带 15 秒超时，异常媒体不再无限冻结 UI。
- `WhisperMaxSegmentSeconds` 现在真正参与分句，长句会按配置的最大时间窗口切分。
- 外部语音 CLI 的 stdout/stderr、进程取消和子进程清理从大型 speech engine 中拆到独立 runner。
- README 增加 1.1 功能、诊断日志和跨平台现状，新增详细的 macOS/Linux 迁移路线文档。
- GitHub Actions 第三方 action 固定到已核对的 commit SHA，减少可移动标签带来的供应链风险。

### 修复

- 修复不同会话重复使用 `speaker-1` / `local-user` 时，后保存会话覆盖较早 speaker 记录的问题。
- 修复合并或删除 speaker/segment 后，SQLite 仍保留旧行的问题。
- 修复 `appsettings.json` 或 `models.json` 损坏时应用在主窗口出现前崩溃的问题；现在会备份为 `.broken-*`、恢复安全默认值并提示用户。
- 修复异步 UI command 异常可能升级为 WPF 未处理异常的问题。
- 修复实时识别慢于输入时音频事件闭包和 PCM 缓冲无限排队的问题。
- 修复仅启动后关闭也会写入零片段空会话、长期污染历史列表的问题；已存在会话即使删空仍会正确保存删除结果。
- 修复实时模型初始化期间其他命令可能被过早启用，以及录音停止与最后音频回调竞争时可能重建已释放 WAV writer 的问题。
- 修复精修保存失败时内存稿可能先于数据库切换，以及不可写诊断目录可能反过来阻止应用启动的问题。
- 修复 Core/STT 测试中写死 `cmd.exe` 与反斜杠路径、导致 Linux 测试失败的问题。
- 修复从非程序目录启动时，相对数据库、录音、导出和日志路径可能跟随启动进程工作目录的问题。

### 安全与发布

- 模型归档提取在压缩包成员元数据和实际展开文件两层限制大小，降低异常压缩膨胀耗尽磁盘的风险。
- NuGet 全依赖漏洞审计继续作为 CI 与 Release 阻断门禁。
- Windows 正式包仍保持模型权重、用户配置、运行数据与 PDB 为 0，并要求 9 份第三方许可原文齐全。
- 版本入口提升为 `1.1.1`；正式 tag 工作流会继续统一 App、Core、Updater 与安装器版本。

## [1.0.1] - 2026-07-17

### 新增

- 新增简体中文与 English 界面切换，首次启动、设置、模型页面、更新提示和安装器均支持双语。
- 新增首次启动引导，可在引导中选择、下载并启用所需语音模型。
- 新增应用内设置页面，不再单独打开设置窗口。
- 新增 GitHub Release 更新机制：启动后自动检查，也可在设置中手动检查、阅读更新说明并安装更新。
- 新增 Windows x64 安装器、便携 ZIP 和对应 SHA256 校验文件。

### 改进

- 模型页面明确分隔 Offline、Realtime 与 Feature Resources，并显示用途、语言、大小、下载状态和缺失模型提示。
- 下拉列表支持点击整个控件展开；较长的 Realtime 模型名称和说明可以换行显示。
- 优化窗口控制按钮、模型卡片、设置按钮和圆角区域的尺寸、对齐、颜色与点击范围。
- Speaker 标签改为按现有 Speaker 列表逐项编辑；没有 Speaker 时不再显示无效输入框。
- 主界面右上角显示当前版本，便于区分正式版与开发构建。

### 修复

- 修复首次引导进入第二步时可能卡死无响应的问题。
- 修复缺少模型时提示不准确或未弹出的问题。
- 修复模型页文字区域、标签和按钮周围出现透明边、白线或竖线的问题。
- 修复英文引导仍生成中文默认会议标题，以及 Whisper 多语言标签重复显示的问题。
- 修复设置页关闭后仍残留语言事件订阅的问题。
- 修复 Silero VAD 旧下载地址返回 HTTP 401，导致 Realtime 模型下载失败的问题。

### 安全与发布

- 发布包不包含任何模型权重、用户配置、录音、导出数据或 PDB 调试符号；模型由用户安装后自行下载。
- 模型下载强制使用 HTTPS、固定 SHA256、临时文件和有限重试，并加强归档成员路径与文件大小检查。
- 更新器校验 Release 资产、版本、SHA256、下载路径和解压目标，降低损坏包与路径穿越风险。
- Release 流程会阻止依赖漏洞、版本不一致、模型权重、用户数据、PDB 或第三方许可缺失的构建。
- 随安装器和便携包提供 9 份第三方许可原文。
